# Scenario: Community Onboarding

## Target
Create contributor documentation to lower the barrier for open-source adoption — the project's #1 strategic priority (developer experience).

## Current State
- Getting-started guide exists (developer onboarding for module creation)
- No CONTRIBUTING.md (how to contribute to the platform itself)
- No architecture decision records (ADRs) explaining design choices
- No code of conduct

## Desired State
Contributors can understand the project, find where to help, and submit quality contributions without needing to reverse-engineer conventions from code.

## Success Criteria
1. CONTRIBUTING.md exists with clear workflow
2. Architecture overview accessible to new contributors
3. Code of conduct in place
4. Links from README or docs index

## Scope
- Pages affected: None
- Components touched: None
- Data changes: None
- Risk level: Low
